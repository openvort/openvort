# OpenVort 产品规划管理

## 目录

所有产品规划存放在 `.cursor/plans/` 目录下，每个重大功能模块一个 `.md` 文件。

## 已有规划

| 文件 | 模块 | 状态 | 创建日期 |
|------|------|------|----------|
| `vortgit.md` | VortGit — Git 仓库管理插件 | Phase 3 已完成，Phase 4 部分完成 | 2026-03-01 |
| `ai-coding.md` | AI 编码能力 — 对话驱动的代码修改与环境生成 | Phase 3a + 3b 已完成 | 2026-03-01 |

## 规划文档规范

### 文件命名
- 使用模块英文名，小写，如 `vortgit.md`、`vortreport.md`
- 一个功能模块一个文件，不合并

### 文档结构
每个 plan 应包含以下章节：
1. **标题** — 模块名 + 一句话描述
2. **状态行** — 当前开发阶段 + 创建日期
3. **架构** — 整体架构图（mermaid）
4. **目录结构** — 后端 + 前端文件布局
5. **数据模型** — 表设计
6. **分阶段实施计划** — Phase 1/2/3/4...

### 状态更新
- 每完成一个 Phase，更新 plan 中的状态行
- 重大设计变更时同步更新 plan

## 操作步骤

### 新增规划
1. 在 `.cursor/plans/` 下创建 `{module}.md`
2. 按上述结构填写内容
3. 更新本 Skill 的「已有规划」表格

### 查看规划
直接读取 `.cursor/plans/` 目录下的对应文件。

### 宣传素材准备
遍历所有 plan 文件，提取各模块的核心亮点和当前状态，可用于：
- README 功能列表
- Release Notes
- 项目官网介绍
- 社区推广文案
