# AI 员工体系设计

## 概念澄清

### 当前系统的问题

系统设计中存在以下概念混淆：

| 概念 | 当前位置 | 问题 |
|------|----------|------|
| **职位 (position)** | `Member.position` | 真人有，AI 员工也可以有，边界模糊 |
| **岗位 (post)** | `Post` 表 | 仅用于 AI 员工，但与职位未区分 |
| **技能** | `Skill` / `MemberSkill` / `PostSkill` | 体系较清晰 |

### 核心问题

1. **职位和岗位没有明确区分** - 两者都是"title"，但含义不同
2. **AI 员工和真人的功能边界模糊** - 都能分配任务、都能触发定时任务
3. **缺少对"AI 代理" vs "AI 员工"两种模式的区分**

## 重新设计：概念分层

### 1. 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     真实组织架构（真人）                          │
│  ┌─────────────┐    ┌─────────────┐                            │
│  │ Department  │───▶│   Member    │ ── position (职位)          │
│  │   (部门)     │    │   (成员)     │ ── is_virtual=False        │
│  └─────────────┘    └─────────────┘                            │
│                              │                                  │
│                              ▼ ReportingRelation                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     AI 员工体系                                  │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │    Post     │───▶│   Member    │◀───│  Reporting  │         │
│  │  (岗位模板) │    │ (AI员工实例) │    │  Relation   │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│         │                   │                   │                │
│         ▼                   ▼                   ▼                │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐           │
│  │  PostSkill  │    │ MemberSkill │    │   汇报     │           │
│  │ (岗位技能)  │    │ (员工技能)  │    │  对象      │           │
│  └─────────────┘    └─────────────┘    └─────────────┘           │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 概念定义

| 概念 | 定义 | 适用对象 |
|------|------|----------|
| **职位 (position)** | 真实组织中的职位头衔 | 真人 |
| **岗位 (post)** | AI 员工角色模板 | AI 员工 |
| **技能 (skill)** | 能力知识单元 | 两者都有 |
| **AI 代理模式** | 真人身份的 AI 辅助 | 真人 |
| **AI 员工模式** | 独立工作的 AI 员工 | AI 员工 |

---

## 关键区分：AI 代理 vs AI 员工

这是设计的核心差异点：

| 特性 | AI 代理（真人模式） | AI 员工模式 |
|------|-------------------|-------------|
| 身份定位 | 真人的附属/助手 | 独立员工 |
| 对话交互 | 人主导，AI 执行建议 | AI 自主，主动汇报 |
| 创建任务 | 需要人确认 | AI 自行创建 |
| 汇报机制 | 无 | 定时向真人汇报 |
| 工具权限 | 有限（辅助工具） | 完整（独立工作） |

### 对话路由区分

```
对话目标选择:
├── 发起与真人对话
│   └── target_type="member", target_id=真人ID
│       → AI 代理模式（辅助工具，决策需人确认）
│
└── 发起与 AI 员工对话
    └── target_type="virtual_member", target_id=AI员工ID
        → AI 员工模式（独立工作，主动汇报）
```

## 数据模型设计

### 1. Member 表优化建议

```python
class Member(Base):
    # 通用字段
    name: Mapped[str]
    avatar_url: Mapped[str]
    
    # 真人专属
    position: Mapped[str] = ""        # 职位（title）
    is_account: Mapped[bool]          # 是否有登录账号
    
    # AI 员工专属
    is_virtual: Mapped[bool]           # 是否 AI 员工
    post: Mapped[str] = ""            # 岗位模板 key（引用 Post.key）
    persona: Mapped[str] = ""          # 人设（覆盖 post.default_persona）
    auto_report: Mapped[bool]          # 是否自动汇报
    report_to: Mapped[str] = ""        # 汇报对象 member_id（新增字段）
    
    # 技能（通用）
    skills: Mapped[str] = "[]"         # JSON 技能ID列表
```

### 2. 新增 AI 代理模式支持（可选）

```python
class ReportingRelation(Base):
    reporter_id: Mapped[str]           # 下属
    supervisor_id: Mapped[str]         # 上司
    relation_type: Mapped[str]         # "direct" / "dotted" / "functional"
    
    # 新增字段
    is_ai_agent: Mapped[bool] = False  # reporter 是否为真人的 AI 代理
    ai_agent_for: Mapped[str] = ""      # 代理的真实成员 ID
```

## 产品功能映射

### 真人功能

- [ ] 职位管理（来自组织架构同步或手动设置）
- [ ] AI 代理模式聊天（辅助工具风格）
- [ ] 给 AI 员工分配任务
- [ ] 接收 AI 员工汇报

### AI 员工功能

- [ ] 岗位模板管理（Post / PostSkill）
- [ ] 独立员工模式聊天（自主工作风格）
- [ ] 创建个人任务
- [ ] 定时汇报（向 report_to 汇报）
- [ ] 技能配置（MemberSkill）

---

## 总结

设计的核心原则：

1. **职位 vs 岗位**：
   - 职位 = 真实组织中的 title（真人）
   - 岗位 = AI 员工角色模板（AI 员工）

2. **AI 代理 vs AI 员工**：
   - AI 代理 = 真人的"第二大脑"，执行但不确定
   - AI 员工 = 独立员工，自主工作并汇报

3. **汇报关系**：
   - AI 员工必须明确 `report_to`（向谁汇报）
   - 这才是真人和 AI 员工的核心区别
