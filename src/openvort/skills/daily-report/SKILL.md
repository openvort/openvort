## 日报生成工作流

当用户要求生成日报（如"帮我写日报"、"发日报"、"生成今天的日报"）时，执行以下流程。

### 流程一：自动采集（有系统工作记录的用户）

1. 调用 `report_submit` 的 `generate` 动作，传入用户姓名、`report_type="daily"`、当天日期
2. 检查返回结果中的 `has_system_data`：
   - **true**：系统已自动采集到 VortFlow 任务/缺陷数据或 Git 提交记录
   - **false**：转入流程二
3. 基于 `collected_data` 和草稿内容，用自然语言重新组织汇报：
   - 将任务和缺陷标题整合为可读的工作描述
   - 关联代码提交和任务完成情况
   - 询问用户是否有"遇到的问题"或"明日计划"需要补充
4. 展示给用户确认，等待用户回复
5. 用户确认后，调用 `report_submit` 的 `submit` 动作（传入 `report_id`）提交

### 流程二：引导式（无系统记录的用户）

1. 引导用户描述工作内容："请简单告诉我你今天主要做了哪些工作，我来帮你整理成日报。"
2. 根据用户描述整理成日报格式：
   - 今日工作（按重要性排列）
   - 遇到的问题（如有）
   - 明日计划（如有，否则询问）
3. 展示给用户确认，确认后调用 `report_submit` 的 `submit` 动作提交

### 定时推送

当用户要求设置"每天自动提醒写日报"或"定时生成日报"时，使用 `schedule_manage` 工具创建定时任务：
- 调度方式：cron
- 建议时间：每个工作日 17:30（`30 17 * * 1-5`）
- prompt 示例："请帮我生成今天的日报并发送给我确认"
- 设置 `target_member_id` 为目标用户的 member_id

### 交互规则

- 用户说"确认"、"可以"、"提交"、"没问题"时 → 立即提交
- 用户说"改一下 xxx"或提出修改意见 → 修改内容后重新展示
- 用户补充了"遇到的问题"或"明日计划" → 整合到日报内容中
- 日报内容应简洁明了，突出关键产出，不暴露内部 ID
